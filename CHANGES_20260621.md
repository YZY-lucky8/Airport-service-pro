## 修复的问题

### 🔴 致命 Bug 修复
| # | 问题 | 修复方案 |
|---|------|---------|
| 1 | 无后端认证，所有管理端 API 可被任何人调用 | 新增 `middleware/auth.js`，纯 Node.js JWT 实现（零外部依赖），HMAC-SHA256 签名 |
| 2 | 没有 MySQL 时项目完全无法启动 | 新增 `backend/db-adapter.js`，MySQL 优先，自动降级 SQLite，对外暴露统一 pool 接口 |
| 3 | SQLite 模式下 `pool` 对象初始化时序问题 | 使用 `Object.defineProperty` getter 导出 pool，确保异步初始化后 API 路由可正确读取 |
| 4 | SQLite adapter 返回三层嵌套数组 | 修复为 `[[rows]]`，与 mysql2 的 `pool.query` 返回格式一致 |
| 5 | SQLite adapter `flight` 表 INSERT 少一个 `?` | 8 列配 8 个占位符 |
| 6 | `app.js` 结构被破坏——`app.post()` 在 `const app = express()` 之前调用 | 子任务+手动重写 app.js，统一结构 |
| 7 | `app.js` 中 `require('./middleware/auth')` 路径错误 | 修正为 `require('../middleware/auth')` |

### 🟡 安全加固
| 项目 | 说明 |
|------|------|
| JWT 认证中间件 | 管理端全部 API 加了 `authMiddleware`（admin/terminals/alerts/usage-stats/faq-stats/call-records/attack-status/whitelist/blacklist/defense-logs/audit-logs/audit/report） |
| 公开 API 保持不认证 | /api/health/db, /api/auth/login, /api/agent/*, /api/token/generate, /api/check-in, /api/log, /api/stats, /api/logs |
| IP 黑名单持久化 | Bloom Filter 启动时从 `ip_blacklist` 数据库表加载 |
| IP 白名单 | 从数据库 `ip_whitelist` 表读取，白名单 IP 跳过限流 |
| CSRF Token API | 新增 `/api/csrf-token` 接口 |
| 管理员管理 API | 新增 add-admin / remove-admin / change-password / admins 列表 |
