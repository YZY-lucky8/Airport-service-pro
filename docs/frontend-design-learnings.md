# 前端设计 Skill 学习笔记
# 来源：Anthropic frontend-design + avoid-ai-design + taste-skill
# 2026-06-20

## 核心原则

### 1. 反 AI 流水线设计
- AI 生成的设计聚集在三种默认外观：
  (1) 暖色奶油背景(#F4F1EA) + 高对比衬线字体 + 赤陶色点缀
  (2) 近黑背景 + 酸绿色/朱红色点缀
  (3) 报纸风格 + 极细线条 + 零圆角 + 密集列
- **禁止的默认**：Inter 字体、紫色渐变、shadcn 默认主题、玻璃态效果
- **每个设计必须有一个"签名元素"** — 这个页面被记住的唯一特征

### 2. 适老化设计原则（结合 WCAG）
- 字体最小 16px，标题 24px+
- 按钮触摸区域至少 44x44px
- 对比度至少 4.5:1（AA级）
- 清晰的视觉层次
- 减少不必要的动画

### 3. 设计决策方法
- **两遍工作法**：先制定设计计划，再编码
- **一个方向**：每个页面只做一个大胆的视觉决定
- **约束与自律**：签名元素周围保持简洁

### 4. 字体配对
- 显示字体 + 正文字体 + 工具字体（可选）
- 不要 Inter — 用 Geist、Satoshi、Cabinet Grotesk 等
- 标题 `tracking-tighter`，正文 `leading-relaxed max-w-[65ch]`

### 5. 颜色规则
- 最多 1 个强调色，饱和度 < 80%
- 禁止紫色/蓝色发光（Lila 规则）
- 一个调色板贯穿全站

### 6. 布局多样化
- DESIGN_VARIANCE > 4 时避免居中 Hero
- 使用 CSS Grid 而非 Flexbox 百分比计算
- 使用 `min-h-[100dvh]` 而非 `h-screen`

### 7. 动画原则
- 有目的的动画：页面加载序列、滚动触发、悬停微交互
- 一个精心编排的时刻 > 零散的效果
- 有时少即是多，过度动画反而像 AI 生成的

### 8. 文字写作
- 从终端用户角度写
- 主动语态优先
- 具体胜于机智
- 错误信息解释发生了什么和如何修复
